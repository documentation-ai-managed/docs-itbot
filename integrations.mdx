## Overview

Use webhooks to receive real-time updates, leverage the REST API for custom automation, and export documentation to various formats.

<Callout kind="tip">

</Callout>

## Setting Up Webhooks

Webhooks enable ITBot to push events like doc updates or user actions to your endpoint. Follow these steps to configure one.

<Steps>
  <Step title="Create Webhook" icon="plus">
    In your ITBot dashboard, navigate to Settings > Webhooks > New Webhook.
  </Step>
  <Step title="Add Endpoint URL" icon="link">
    Enter your secure HTTPS endpoint, such as `https://your-webhook-url.com/webhook`.
  </Step>
  <Step title="Select Events" icon="bell">
    Choose events like `doc.updated` or `user.invited`.
  </Step>
  <Step title="Test and Save" icon="check-circle">
    Send a test payload and save the webhook.
  </Step>
</Steps>

Example webhook payload:

<CodeGroup tabs="JSON,Schema">
  ```json
  {
    "event": "doc.updated",
    "doc_id": "doc_123abc",
    "timestamp": "2024-10-15T10:30:00Z",
    "user_id": "user_456def"
  }
  ```

  ```json
  {
    "type": "object",
    "properties": {
      "event": { "type": "string" },
      "doc_id": { "type": "string" },
      "timestamp": { "type": "string", "format": "date-time" },
      "user_id": { "type": "string" }
    }
  }
  ```
</CodeGroup>

## ITBot REST API

Build custom integrations with ITBot's REST API at `https://api.example.com/v1`. Authenticate using Bearer tokens.

<ParamField header="Authorization" param-type="string" required="true">
  Bearer `{YOUR_API_KEY}`.
</ParamField>

<ParamField query="workspace_id" param-type="string" required="true">
  Your ITBot workspace identifier.
</ParamField>

Fetch a document:

<Request tabs="JavaScript,cURL">
  ```javascript
  const response = await fetch('https://api.example.com/v1/docs/doc_123abc?workspace_id=YOUR_WORKSPACE_ID', {
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY'
    }
  });
  const doc = await response.json();
  console.log(doc);
  ```

  ```bash
  curl -X GET "https://api.example.com/v1/docs/doc_123abc?workspace_id=YOUR_WORKSPACE_ID" \
    -H "Authorization: Bearer YOUR_API_KEY"
  ```
</Request>

<Response tabs="200">
  ```json
  {
    "id": "doc_123abc",
    "title": "API Guide",
    "content": "Full document markdown...",
    "updated_at": "2024-10-15T10:30:00Z"
  }
  ```
</Response>

## Exporting Documentation

Export your ITBot docs to share externally or archive.

<Tabs>
  <Tab title="PDF" icon="file-text">
    Use the API to generate PDF exports.
    
    ```bash
    curl -X POST "https://api.example.com/v1/docs/doc_123abc/export?format=pdf" \
      -H "Authorization: Bearer YOUR_API_KEY" \
      --output "guide.pdf"
    ```
  </Tab>
  <Tab title="Markdown" icon="file">
    Download raw Markdown for GitHub or other tools.
    
    ```javascript
    const response = await fetch('https://api.example.com/v1/docs/doc_123abc/export?format=md', {
      headers: { 'Authorization': 'Bearer YOUR_API_KEY' }
    });
    const md = await response.text();
    ```
  </Tab>
  <Tab title="HTML" icon="code">
    Get clean HTML for embedding.
    
    <Callout kind="info">
      HTML exports include styles for immediate use.
    </Callout>
  </Tab>
</Tabs>

<Expandable title="Advanced Webhook Security" default-open="false">
  Always validate payloads with HMAC signatures. ITBot sends a `X-ITBot-Signature` header using your secret.
</Expandable>
